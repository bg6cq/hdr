# 手写数字识别神经网络演示

一个交互式 Web 应用，用于学习神经网络如何识别手写数字。

**纯前端** — 所有神经网络计算在浏览器中运行，无需后端。

## 网络结构

```
输入层 [784]  → 隐藏层1 [128]  → 隐藏层2 [64]  → 输出层 [10]
 28×28 像素      ReLU 激活       ReLU 激活       Softmax
```

总参数量：约 10.9 万（W1: 100,352 | W2: 8,192 | W3: 640 + 偏置）

## 快速使用

1. 点击「🚀 批量训练」，选每数字 50 或 100
2. 等待训练完成（状态栏显示实时准确率与损失）
3. 切换到「识别」模式，画板上写数字，自动预测

也可按 Enter 单步训练（需勾选数字标签）。

## 界面

| 区域 | 内容 |
|------|------|
| **左侧** | 手写画板 + 训练/识别控制面板 |
| **中间** | 网络激活可视化（输入 → 隐藏层1 → 隐藏层2 → 输出层） |
| **右侧** | 权重可视化（W1 / W2 / W3 可切换，每神经元独立归一化） |

### 权重的三层

| 权重 | 连接 | 尺寸 | 说明 |
|------|------|------|------|
| W1 | 输入层 → 隐藏层1 | 128 × 784 | 每个神经元 28×28 像素特征模板（可直接看出笔画轮廓） |
| W2 | 隐藏层1 → 隐藏层2 | 64 × 128 | 隐藏层到隐藏层的连接权重 |
| W3 | 隐藏层2 → 输出层 | 10 × 64 | 每个数字的输出神经元权重，左上角标注数字 |

### 权重图颜色
- **橙色** → 正权重（像素激活该神经元）
- **蓝色** → 负权重（像素抑制该神经元）
- **白色** → ≈ 零（无关）

> 每个小图独立归一化（除以自身最大绝对值），保证弱特征也可见。

## 手写输入预处理

手写笔画经过 **MNIST 标准预处理**后再送网络：

1. 裁剪笔画边界框（去掉周围空白）
2. 等比缩放到 20×20（统一大小）
3. 按质心居中到 28×28（对齐位置）

**原因**：手写位置/大小不固定，网络在 MNIST 训练时看到的所有数字都是严格居中、大小统一的。不做此预处理识别率会从 ~89% 骤降到 ~20%。

## 训练数据

使用 **MNIST 数据集**（5000 张真实手写数字，每数字 500 张）。
像素值以 0-255 整数压缩存储（8.8MB），加载时浏览器自动 /255 还原。

| 每数字数 | 总样本 | 测试准确率（30 epoch） |
|----------|--------|----------------------|
| 50 | 500 | ~89% |
| 100 | 1000 | ~92% |

> 注：训练集上的准确率会更高（过拟合），批量训练结束时显示的是**训练集准确率**。

## 键盘快捷键

| 键 | 功能 |
|----|------|
| Enter | 训练一次（训练模式下） |

## 导出 / 加载

训练好的模型可导出为 JSON 文件，下次加载恢复，无需重新训练。

## 项目背景 — Vibe Coding

这个页面是通过与 AI 的多轮对话逐步成型的（详见页面右上角「📖 说明」）：

1. **初始想法**：手写画板 + 各层网络可视化 + checkbox 标注 + 模式切换
2. **权重图**：能否把权重用图像显示？→ W1 以 28×28 像素模板呈现
3. **MNIST 数据集**：手写训练效果差 → 内置 5000 张 MNIST 真实数据
4. **居中预处理**：识别率低 → 根因是整板降采样未做 MNIST 标准居中预处理 → 修复后识别率从 <20% 恢复到 80%+

### Vibe Coding 关键词

> 我在学习使用神经网络做手写数字识别，想做一个 web 页面展示训练和识别过程。初步设想 web 页面左侧是手写体输入框，可以用鼠标或手写板输入数字，中间是各层的神经网络，右侧是 10 个 checkbox 对应 0-9 数字，神经网络的各层数字可以在某个区域用图片的方式展现，每个点是其中一个参数的颜色转换。其中有个按钮切换训练还是识别模式，训练模式时，左侧输入数字，右侧选择结果，此时进行一次训练。识别模式时，左侧输入数字，右侧出结果。训练的参数可以导出文件，并可以进行加载，这种思路怎么样？
>
> 网络的权重能用一个图显示吗？
>
> 自己训练效果太差。下载 MNIST 数据集，增加一个利用数据集训练的功能。
>
> 训练出来的模型识别率很低。
>
> 注：根因明确：sampleCanvas() 直接整板降采样，没做 MNIST 标准的居中预处理。数字画偏一点，网络就认不出——这正是手写识别率低的元凶。
